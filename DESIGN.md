# ארכיטקטורה מונחית אירועים – מסמך תכנון

## מבנה המחלקות

```
ISubscriber (ממשק)
    └── Subscriber

Event (struct)

Partition
    משתמש ב־ → Event (תור מעגלי)

EventBus
    מחזיק → Partition (מפה)
    מחזיק → ISubscriber* (מפה)
    מחזיק → SubscriptionInfo (מפה מקוננת)

Publisher
    משתמש ב־ → EventBus

Tests
    יוצר → EventBus חדש לכל בדיקה
```

---

## תיאור המחלקות

### `Event` (struct – `Event.h`)
מבנה נתונים פשוט המייצג אירוע בודד.
שדות: `globalId`, `type`, `partitionName`, `data`.
ערך `globalId = -1` משמש כסנטינל (אירוע ריק/לא חוקי).

---

### `Partition` (`Partition.h` / `Partition.cpp`)
מנהלת תור מעגלי בגודל קבוע לשמירת אירועים.

**עיקרון מרכזי:** מיקום האירוע במערך = `globalId % capacity`

| שדה | משמעות |
|---|---|
| `nextGlobalId_` | המזהה שיוקצה לאירוע הבא (עולה תמיד) |
| `buffer_[capacity]` | מערך האירועים; התא `buffer_[id % cap]` מכיל את האירוע עם אותו מזהה |

- `getOldestAvailableId()` = `max(0, nextGlobalId - capacity)`
- `isAvailable(id)` בודק שהמזהה נמצא בתחום `[oldest, nextGlobalId)`
- כאשר התור מלא, אירוע חדש פשוט דורס את הישן – ללא צורך בניהול מצביעים נפרדים.

---

### `ISubscriber` (ממשק – `ISubscriber.h`)
מגדיר את החוזה שכל מנוי חייב לעמוד בו:
- `getName()` – מחזיר את שם המנוי
- `onEvent(Event)` – מטפל באירוע שהתקבל

כל מחלקה שמממשת ממשק זה יכולה להירשם למנגנון.

---

### `Subscriber` (`Subscriber.h` / `Subscriber.cpp`)
מימוש קונקרטי של `ISubscriber`.
מדפיס את האירוע שהתקבל ושומר אותו ברשימה פנימית.
הרשימה הפנימית משמשת את מחלקת הבדיקות לאימות הפלט.

---

### `Publisher` (`Publisher.h` / `Publisher.cpp`)
מייצג רכיב שמייצר אירועים.
עוטף קריאה ל־`EventBus::publish` ומוסיף את שם המפרסם לפלט.
**אינו מחזיק ישירות רשימת מנויים** – כל ניתוב עובר דרך ה־`EventBus`.

---

### `SubscriptionInfo` (struct ב־`EventBus.h`)
מייצג את מצב המנוי עבור מחיצה ספציפית:

| שדה | משמעות |
|---|---|
| `eventType` | סוג האירוע שהמנוי מסנן (מחרוזת ריקה = קבל הכל) |
| `consumptionPointer` | מזהה האירוע הגלובלי שהמנוי יצרוך בפעם הבאה |

---

### `EventBus` (`EventBus.h` / `EventBus.cpp`)
המנגנון המרכזי של המערכת. מחזיק בעלות על המחיצות ועל מצב כל המנויים.

| פעולה | תיאור |
|---|---|
| `createPartition(name, cap)` | יוצרת מחיצה חדשה עם קיבולת נתונה |
| `registerSubscriber(ptr)` | רושם מנוי לפי שמו |
| `subscribe(sub, part, type)` | יוצרת `SubscriptionInfo`; המצביע מתחיל ב־`nextGlobalId` הנוכחי |
| `publish(type, part, data)` | כותב אירוע למחיצה ומדפיס לוג |
| `consume(sub, part)` | מתקדם מהמצביע קדימה, מדלג על אירועים מסוג שגוי, מעביר את הראשון המתאים דרך `onEvent` |
| `rewind(sub, part, steps)` | מזיז את המצביע אחורה; מחזיר שגיאה אם היעד אינו זמין |
| `printStats()` | מציגה מצב כל המחיצות ומצביעי המנויים |

---

### `Tests` (`Tests.h` / `Tests.cpp`)
מריצה 7 בדיקות חובה, כל אחת על `EventBus` נפרד חדש.

## החלטות תכנון מרכזיות

### 1. תור מעגלי ללא ניהול ראש/זנב
במקום לשמור מצביע ראש ומצביע זנב, משתמשים בנוסחה `slot = id % capacity`.
הגיל הזמין מחושב תמיד: `oldest = max(0, nextId - capacity)`.
תוצאה: קוד פשוט ביותר, אפס ניהול מצביעים.

### 2. מודל משיכה (Pull)
המנויים קוראים ל־`consume()` באופן פעיל – האירועים אינם נדחפים אוטומטית.
זה מה שמאפשר את מנגנון החזרה לאחור: המיקום שמור לכל מנוי ומתעדכן רק כשהוא מחליט לצרוך.

### 3. מצביע צריכה גלובלי (לפי `globalId`)
המצביע מצביע על מזהה מוחלט בתוך הסדרה הגלובלית של המחיצה, לא על אינדקס מסנן.
זה הופך את ה־`rewind` לאינטואיטיבי: "חזור N צעדים" = `pointer -= N`.

### 4. סינון סוג האירוע בזמן הצריכה
`consume()` מתקדם דרך כל הסלוטים ומדלג על אירועים מסוג שגוי.
המצביע מתקדם על כל סלוט – גם על אלה שדולגו – כך שה־`rewind` עובד תמיד על מיקומים מוחלטים.

### 5. טיפול בדריסה (Overwrite)
- **`consume`**: אם המצביע הצביע על אירוע שנדרס, מדפיס אזהרה ומקפץ אוטומטית ל־`oldestAvailableId`.
- **`rewind`**: אם היעד קטן מ־`oldest`, מחזיר מחרוזת שגיאה ברורה (אינו זורק חריגה).

### 6. הפרדת אחריות
| מחלקה | אחריות |
|---|---|
| `Partition` | אחסון בלבד |
| `EventBus` | ניתוב, מנויים, מצביעים |
| `Publisher` | ממשק פרסום בלבד |
| `Subscriber` | טיפול ואחסון אירועים שהתקבלו |
| `Tests` | אימות בלבד |

## הוראות הרצה

### דרישות
- מהדר C++17 (g++ גרסה 10 ומעלה, או MSVC 2019+)

### קומפילציה עם g++
```bash
g++ -std=c++17 -Wall -o EventDriven \
    main.cpp Partition.cpp Subscriber.cpp Publisher.cpp EventBus.cpp Tests.cpp
```

### קומפילציה עם CMake
```bash
cmake -B build .
cmake --build build
./build/EventDriven
```

### הרצה
```
./EventDriven        # Linux / Mac
.\EventDriven.exe    # Windows
```

התוכנית מריצה תחילה את תרחיש ההדגמה המלא, ולאחר מכן את 7 בדיקות החובה.
